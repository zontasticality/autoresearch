# Research Sessions — Shared Configuration

This directory (`~/Research/`) contains research session subdirectories and
downloaded source files (PDFs, EPUBs, etc.).

## Directory Layout

```
~/Research/
├── CLAUDE.md                          ← this file
├── .session-template.md               ← session CLAUDE.md template
├── config.json                        ← user config (gitignored)
├── config.example.json                ← config template
├── .mcp.json                          ← MCP server config (gitignored)
├── .mcp.json.example                  ← MCP config template
├── setup.sh                           ← first-time setup script
├── .bin/
│   ├── research                       ← single entry point (Python, argparse)
│   ├── annas-mcp                      ← pre-built binary (git LFS)
│   ├── annas-mcp-src/                 ← annas-mcp source (git submodule)
│   └── lib/
│       ├── __init__.py
│       ├── sources.py                 ← shared: load/save, formatting, templates
│       ├── enrich.py                  ← CrossRef enrichment
│       ├── pdf2html.py                ← PDF conversion + HTML post-processing
│       ├── fragment.py                ← text fragment URL generation
│       ├── render.py                  ← generate sources.md from sources.json
│       ├── sync.py                    ← git sync to GitHub
│       └── session.py                 ← new/list/resume/open session management
├── Some Book - Author 2024.pdf        ← shared downloads (from anna-mcp)
├── Another Book - Author 2019.epub
├── 2026-02-25-topic-slug/             ← session directory
│   ├── CLAUDE.md                      ← session-specific instructions
│   ├── sources.json                   ← structured source of truth
│   ├── sources.md                     ← auto-generated from sources.json
│   ├── notes.md
│   ├── summary.md
│   ├── questions.md
│   └── docs/                          ← HTML papers (auto-generated by pdf2html)
└── 2026-02-26-another-topic/
    └── ...
```

## Setup

Run `./setup.sh` after cloning. It creates `config.json` (GitHub org, email)
and `.mcp.json` (annas-mcp secret key, paths). See `config.example.json` and
`.mcp.json.example` for the expected format.

## Sources JSON Protocol

`sources.json` is the single structured source of truth for each research
session. It tracks all sources (downloaded and web-only), their metadata,
reading status, and relevance assessments. It replaces the old
`downloads.manifest.md` file.

### Schema Overview

```json
{
  "session": {
    "title": "Session Title",
    "date": "YYYY-MM-DD",
    "question": "Research question",
    "github_repo": "<github-org>/<session-slug>",
    "pages_url": "https://<github-org>.github.io/<session-slug>"
  },
  "sources": [
    {
      "id": 1,
      "title": "Paper Title",
      "authors": ["First Author", "Second Author"],
      "year": 2024,
      "doi": "10.xxxx/...",
      "pmid": "12345678",
      "journal": "Journal Name",
      "volume": "1", "issue": "2", "pages": "10-20",
      "type": "journal-article",
      "found_via": "How this source was discovered",
      "download": {
        "hash": "<md5 from anna-mcp>",
        "format": "pdf",
        "filename": "Paper Title.pdf"
      },
      "status": "scouted-only",
      "status_detail": null,
      "relevance_level": "HIGH",
      "relevance": "One-line assessment",
      "html_filename": null,
      "crossref_enriched": false
    }
  ]
}
```

Key fields:
- `download` is `null` for web-only sources. Non-null means restorable via anna-mcp.
- `html_filename` is auto-set by `research pdf2html` after conversion.
- `status`: one of `not-yet-read`, `scouted-only`, `read-partial`, `read-full`, `downloaded`.
- `relevance_level`: one of `LOW`, `MODERATE`, `MODERATE-HIGH`, `HIGH`, `VERY HIGH`, or `null`.
- `crossref_enriched`: whether metadata was auto-populated from CrossRef.

### After Downloading a Source

1. Add an entry to `sources.json` with `download.hash`, `download.format`,
   `download.filename`, and DOI if known.
2. Optionally run `research enrich --add <DOI>` to auto-create the entry.
3. Annotate: `status`, `relevance`, `relevance_level`, `found_via`.

### Restoring Downloads

When the user asks to "restore downloads" in a session directory:

1. Read `sources.json`
2. For each entry where `download` is not null, call
   `mcp__annas-mcp__book_download` with:
   - `hash`: `download.hash`
   - `title`: `download.filename` (without extension)
   - `format`: `download.format`

## Tools (`research` CLI)

All tools are accessed via the `research` command (installed at
`~/.local/bin/research`, delegates to `~/Research/.bin/research`).

### `research enrich` — Add/enrich sources by DOI

```
research enrich [--add] <DOI> [sources.json]    # enrich or add single entry
research enrich --all [sources.json]             # enrich all un-enriched entries
```

Uses the CrossRef API to auto-populate title, authors, year, journal,
volume, issue, pages, and type. Only fills null/missing fields (never
overwrites manual entries). Rate-limits to 1 req/sec.

### `research pdf2html` — Convert PDFs to HTML

```
research pdf2html [--force] [session-dir]
```

Reads `sources.json`, converts PDFs via podman + pdf2htmlEX, post-processes
to fix lazy-loading (strips `<script>` tags, adds `opened` class), fixes
scroll containers (makes body the scroll root for text fragment support),
and injects metadata bars. Updates `sources.json` with `html_filename` and
generates `docs/index.html`.

**GitHub Pages setup** (one-time per repo): after first push with `docs/`,
enable Pages from `main:/docs`:
```bash
gh api repos/<github-org>/<repo>/pages -X POST \
  -f source.branch=main -f source.path=/docs
```

### `research fragment` — Generate text fragment URLs

```
research fragment <html-file> "<quote>" [sources.json]
```

Parses pdf2htmlEX output, extracts text from `<div class="t">` elements,
handles ligatures (fi, fl, ffi, etc.) and whitespace normalization. Uses
range syntax (`text=prefix,suffix`) for multi-div matches.

Auto-detects `sources.json` in the same or parent directory for
`session.pages_url`.

### `research sources` — Render sources.json to markdown

```
research sources [sources.json]
```

Writes `sources.md` in the same directory. Groups downloaded papers first,
then web-only sources. Includes HTML view links, DOIs, status, relevance.

### `research sync` — Sync to GitHub

```
research sync [session-dir]
```

Initializes git, writes `.gitignore`, auto-generates `sources.md` from
`sources.json`, commits all trackable files, creates the GitHub repo if
needed, and pushes to `origin/main`. Binary files are gitignored — only
`.md`, `.json`, `docs/`, and config are committed. The `sources.json` file
allows anyone to reconstruct the full set of downloaded sources.

### Session Management

```
research <question>                    # create session + launch claude
research list|ls                       # list past sessions
research resume|r [query]              # resume a session (by number or name)
research open|o [query]                # print session path
```
